# Task 03 — Core agent pipeline

## Goal
Avoma webhook fires → transcript parsed by Claude → email generated by Claude → Gmail draft created → Slack DM sent. Full pipeline working end-to-end with one hardcoded test PM.

## Done when
- [ ] `POST /webhook/avoma` returns 200 within 2 seconds
- [ ] Background thread processes the full pipeline
- [ ] Draft appears in test PM's Gmail inbox addressed to correct client
- [ ] Slack DM sent to test PM
- [ ] Record saved to `draft_history` table

## Reference
- `docs/integrations.md` → Avoma, Gmail, Slack, Claude sections
- `docs/database.md` → `draft_history` table
- `docs/prompt-design.md` → both prompt structures
- `docs/architecture.md` → data flow: transcript → draft

## Build order within this task

### Step 1: Webhook receiver (`webhook/avoma.py`)
- Verify HMAC-SHA256 signature using `AVOMA_WEBHOOK_SECRET`
- Return 200 immediately
- Spawn `threading.Thread` for `_process_meeting(payload)`
- Log errors to console (Sentry later)

### Step 2: Context extractor (`agent/extractor.py`)
- Takes: transcript string + meeting metadata dict
- Calls Claude API — see `docs/prompt-design.md` Call 1
- Parses JSON response (strip code fences first)
- Falls back to Avoma attendee list if `client_email` empty
- Returns: context dict

Test this in isolation first:
```python
# test_extractor.py
from agent.extractor import extract_context
result = extract_context(SAMPLE_TRANSCRIPT, SAMPLE_METADATA)
print(result)
```
Use the sample transcript from `docs/prompt-design.md`.

### Step 3: Prompt builder (`agent/prompt_builder.py`)
- Takes: pm_id + meeting_type
- Fetches style samples from DB (see `docs/prompt-design.md` → sample selection logic)
- Builds system prompt string
- Builds user prompt string
- Returns: (system_prompt, user_prompt) tuple

For this task: hardcode a fake PM with 2 sample emails in a test fixture. Real PM samples come in Task 05.

### Step 4: Generator (`agent/generator.py`)
- Takes: pm_id + transcript + context dict
- Calls `prompt_builder` to get prompts
- Calls Claude API — see `docs/prompt-design.md` Call 2
- Returns: email string

### Step 5: Gmail integration (`integrations/gmail.py`)
- Takes: pm_id + to_email + to_name + subject + body
- Gets credentials via `auth/google_oauth.py` → `get_credentials_for_pm(pm_id)`
- Creates Gmail draft using `users.drafts.create`
- Returns: gmail_draft_id string

Implement `get_credentials_for_pm` in `auth/google_oauth.py`:
- Fetch token record from DB
- Decrypt refresh token
- Build `Credentials` object
- Refresh if expired
- Return credentials

### Step 6: Slack notification (`integrations/slack.py`)
- Takes: slack_user_id + client_name + meeting_type
- Sends block kit message with "Open Gmail Drafts" button
- See `docs/integrations.md` → Slack section for message structure

### Step 7: Wire it all together (`webhook/avoma.py` → `_process_meeting`)
Flow:
1. Look up PM by `host_email` → exit if not found or `onboarding_complete = false`
2. Call `extract_context`
3. Call `generate_email`
4. Call `create_gmail_draft`
5. Call `save_draft` to DB
6. Call `notify_pm` on Slack
